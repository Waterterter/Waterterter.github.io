0x1.Java特点

1、跨平台特性，也被称作平台无关性，简单来说就是兼容各个操作系统。Sun公司的设计宗旨为“write once， run any where”，支撑这个特性的技术是Java虚拟机技术，简称JVM。这东西貌似是大厂必考察内容。

  • 1、JVM是一套支持Java语言运行的软件系统，定义了指令集、寄存器集、类文件结构栈、垃圾收集栈、内存区域等，提供了跨平台能力的基础框架，jvm运行在os上。
  • 2、Java虚拟机执行Java程序的过程如下图 , Java源程序 .java文件经编译生成.class文件即字节码文件，也称类文件，由jvm解释执行，解释一句执行一句。.class文件这东西是字节码，主要是被虚拟机处理的。

  • 3、JVM执行程序的三个特点：
    1. 动态性。管理功能全部采用的是动态方式。比如说动态分配数组内存、动态链接db、动态new一个对象。
    2. 异常处理。可靠的处理机制。
    3. 多线程。

2、完全面向对象和简单性，Java面向对象是改进c++由来的，语法区别有：

  • 不支持全局变量和宏替换，使用最终变量代替宏替换。避免全局变量和宏替换的副作用（不理解）
  • 数据类型固定长度，与平台无关
  • 类型转换时会检查
  • 不能goto
  • 不支持pointer，使用引用
  • 不支持struct，用class
  • 不支持header，用import导包
  • 内存动态存储且自动管理，动态申请数组和对象的存储空间，自动释放空间

对oop的改进有

  • 为八种基本数据类型提供包装类，使数据类型和类相关联，体现了完全面向对象
  • 将数组设计为引用类型，每个数组由长度属性
  • 不支持面向过程设计，无全局函数，成员方法内不支持static声明局部变量
  • 不支持友元类和运算符重载
  • 单继承。

简单性的体现，无指针用下标，用引用。构造方法采用重载来区别，而不用默认参数。

3、可靠性。Java在语言和运行提供程序运行稳定性和可靠性保证。

  • 语言。规则检查严密，在编译和运行时严格检查，提供异常处理机制，即使在运行时出错也不要怕。
  • 运行架构。提供资源回收机制，对内存资源进行自动管理，跟踪程序使用的所有内存资源。这一切都是Java自己做的。

4、安全性。采用域管理方式，对访问策略配置。
5、多线程。支持多线程。
6、支持分布式网络应用
